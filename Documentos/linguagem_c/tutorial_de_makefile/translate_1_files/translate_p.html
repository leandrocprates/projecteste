Makefiles


A tutorial by example Um tutorial por exemplo

Compiling your source code files can be tedious, specially when you want to include several source files and have to type the compiling command everytime you want to do it. Compilando seu código-fonte arquivos podem ser enfadonho, especialmente quando você deseja incluir vários arquivos de origem e ter de digitar o comando compilar você sempre quis fazer.
Well, I have news for you… Your days of command line compiling are (mostly) over, because YOU will learn how to write Makefiles. Bem, tenho novidades para você ... Seus dias de linha de comando compilação são (principalmente) sobre, porque você vai aprender a escrever Makefiles.
Makefiles are special format files that together with the make utility will help you to automagically build and manage your projects. Makefiles são arquivos formato especial que, juntamente com a marca utilitário irá ajudá-lo a automagicamente construir e gerir os seus projectos.

For this session you will need these files: Para esta sessão vai precisar desses arquivos:

  • main.cpp main.cpp
  • hello.cpp hello.cpp
  • factorial.cpp factorial.cpp
  • functions.h functions.h

I recommend creating a new directory and placing all the files in there. Eu recomendo criar um novo diretório e colocar todos os arquivos lá dentro.

note: I use g++ for compiling. nota: eu uso g + + para compilar. You are free to change it to a compiler of your choice Você é livre para alterá-lo para um compilador de sua escolha

The make utility O utilitário fazer

If you run Se você executar 

 make fazer

this program will look for a file named makefile in your directory, and then execute it. este programa irá procurar por um arquivo chamado makefile no seu diretório e, em seguida, executá-lo.

If you have several makefiles, then you can execute them with the command: Se você tem vários makefiles, então você pode executá-los com o comando: 

 make -f MyMakefile make-f MyMakefile

There are several other switches to the make utility. Existem vários outros parâmetros para o make utilitário. For more info, man make . Para mais informações, o man make

Build Process Build Process

  1. Compiler takes the source files and outputs object files Compiler tem os arquivos de origem e saídas ficheiros objecto
  2. Linker takes the object files and creates an executable Linker tem arquivos objeto e gera um arquivo executável

Compiling by hand Compilando em mão

The trivial way to compile the files and obtain an executable, is by running the command: A maneira trivial de compilar os arquivos e obter um executável, está correndo o comando: 

 g++ main.cpp hello.cpp factorial.cpp -o hello g + + main.cpp-o hello.cpp factorial.cpp Olá

The basic Makefile A base Makefile

The basic makefile is composed of: O makefile básica é composta de: 

 
 target: dependencies alvo: Dependências 
 [tab] system command [guia] sistema de comando


This syntax applied to our example would look like: Esta sintaxe aplicado ao nosso exemplo, seria algo parecido com: 

 all: todos: 
	 g++ main.cpp hello.cpp factorial.cpp -o hello g + + main.cpp-o hello.cpp factorial.cpp Olá

To run this makefile on your files, type: Para executar este makefile em seus arquivos, digite: 

 make -f Makefile-1 make-f Makefile-1

On this first example we see that our target is called all . Quanto a este primeiro exemplo, vemos que o nosso objectivo é chamado de todos. This is the default target for makefiles. Este é o padrão-alvo para makefiles. The make utility will execute this target if no other one is specified. O utilitário irá fazer executar esta meta se não houver um outro seja especificado.
We also see that there are no dependencies for target all , so make safely executes the system commands specified. Também veja se não há dependências alvo para todos, por isso faça com segurança o sistema executa comandos especificados.

Finally, make compiles the program according to the command line we gave it. Por último, faça compila o programa de acordo com a linha de comando que lhe deram.

Using dependencies Usando dependências

Sometimes is useful to use different targets. Às vezes, é útil recorrer a diferentes alvos. This is because if you modify a single file in your project, you don’t have to recompile everything, only what you modified. Isto porque se você modificar um único arquivo em seu projeto, você não tem que recompilar tudo, apenas o que você modificou.
Here is an example: Aqui está um exemplo: 

 all: hello todos: Olá 

 hello: main.o factorial.o hello.o Olá: main.o factorial.o hello.o 
	 g++ main.o factorial.o hello.o -o hello g + + main.o-o factorial.o hello.o Olá 

 main.o: main.cpp main.o: main.cpp 
	 g++ -c main.cpp g + +-c main.cpp 

 factorial.o: factorial.cpp factorial.o: factorial.cpp 
	 g++ -c factorial.cpp g + +-c factorial.cpp 

 hello.o: hello.cpp hello.o: hello.cpp 
	 g++ -c hello.cpp g + +-c hello.cpp 

 clean: limpo: 
	 rm -rf *o hello rm-rf * o Olá

Now we see that the target all has only dependencies, but no system commands. Agora vemos que o alvo só tem todas as dependências, mas não o sistema de comandos. In order for make to execute correctly, it has to meet all the dependencies of the called target (in this case all ). A fim de fazer para executar corretamente, ele tem de satisfazer todas as dependências do chamado alvo (neste caso, todos).

Each of the dependencies are searched through all the targets available and executed if found. Cada uma das dependências são pesquisados através de todos os alvos disponíveis e executada se encontrado.

In this example we see a target called clean . Neste exemplo, vemos um alvo chamado limpa. It is useful to have such target if you want to have a fast way to get rid of all the object files and executables. É útil ter essa meta se você quiser ter uma maneira rápida de se livrar de todos os arquivos objeto e executáveis.

Using variables and comments Usando variáveis e comentários

You can also use variables when writing Makefiles. Você também pode usar variáveis ao escrever Makefiles. It comes in handy in situations where you want to change the compiler, or the compiler options. Vem a calhar em situações em que você deseja alterar o compilador, ou as opções de compilação. 

 # I am a comment, and I want to say that the variable CC will be # Sou um comentário e quero dizer que a variável CC será 
 # the compiler to use. # Para usar o compilador. 
 CC=g++ CC = g + + 
 # Hey!, I am comment number 2. # Ei!, Estou comentário número 2. I want to say that CFLAGS will be the Eu quero dizer que será o CFLAGS 
 # options I'll pass to the compiler. # Opções vou passar para o compilador. 
 CFLAGS=-c -Wall CFLAGS =- c-Wall 

 all: hello todos: Olá 

 hello: main.o factorial.o hello.o Olá: main.o factorial.o hello.o 
	 $(CC) main.o factorial.o hello.o -o hello $ (CC)-o main.o factorial.o hello.o Olá 

 main.o: main.cpp main.o: main.cpp 
	 $(CC) $(CFLAGS) main.cpp $ (CC) $ (CFLAGS) main.cpp 

 factorial.o: factorial.cpp factorial.o: factorial.cpp 
	 $(CC) $(CFLAGS) factorial.cpp $ (CC) $ (CFLAGS) factorial.cpp 

 hello.o: hello.cpp hello.o: hello.cpp 
	 $(CC) $(CFLAGS) hello.cpp $ (CC) $ (CFLAGS) hello.cpp 

 clean: limpo: 
	 rm -rf *o hello rm-rf * o Olá

As you can see, variables can be very useful sometimes. Como você pode ver, as variáveis pode ser muito útil às vezes. To use them, just assign a value to a variable before you start to write your targets. Para utilizá-los, basta atribuir um valor a uma variável antes de começar a escrever os seus objectivos. After that, you can just use them with the dereference operator $(VAR). Depois disso, você pode simplesmente usá-los com o operador dereference $ (VAR).

Where to go from here Para onde ir a partir daqui

With this brief introduction to Makefiles, you can create some very sophisticated mechanism for compiling your projects. Com esta breve introdução aos Makefiles, você pode criar algumas muito sofisticado mecanismo de compilação de seus projetos. However, this is just a tip of the iceberg. No entanto, esta é apenas uma ponta do iceberg. I don’t expect anyone to fully understand the example presented below without having consulted some Make documentation (which I had to do myself) or read pages 347 to 354 of your Unix book. Não espero ninguém compreender plenamente o exemplo apresentado a seguir, sem ter consultado alguns Faça documentação (que eu tinha de fazer eu mesmo) ou ler páginas 347 a 354 do seu livro Unix. 

 CC=g++ CC = g + + 
 CFLAGS=-c -Wall CFLAGS =- c-Wall 
 LDFLAGS= LDFLAGS = 
 SOURCES=main.cpp hello.cpp factorial.cpp FONTES = main.cpp hello.cpp factorial.cpp 
 OBJECTS=$(SOURCES:.cpp=.o) OBJECTS = $ (FONTES:. Cpp =. o) 
 EXECUTABLE=hello Executável = Olá 

 all: $(SOURCES) $(EXECUTABLE) todos: $ (SOURCES) $ (executável) 

 $(EXECUTABLE): $(OBJECTS) $ (Executável): $ (OBJETOS) 
	 $(CC) $(LDFLAGS) $(OBJECTS) -o $@ $ (CC) $ (LDFLAGS) $ (OBJECTS)-o $ @ 

 .cpp.o: . cpp.o: 
	 $(CC) $(CFLAGS) $< -o $@ $ (CC) $ (CFLAGS) $ <-o $ @
