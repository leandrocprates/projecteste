Como usar o dbunit-plugin

Introdução

Nessa página você verá como usar o DBUnit-Plugin, um plugin que permite usar datasets (pense nelas como fixtures para os testes que usam bancos de dados) do DBUnit no Grails.

O plugin não se propõe a cobrir todos os pontos levantados na proposta apresentada na wiki do Grails, mas usa algumas das idéias descritas lá.

Como usar?

Antes de tudo, instale (dooh). O plugin está preparado para lidar com datasets escritas como flat xml, já que essa é a abordagem que considero mais simples. Vejamos alguns exemplos:

Usando apenas um dataset para o teste

Considere a classe abaixo: 

class Person {
        Long id
        String name
        String email
        static constraints = {
                name(blank:false)
                email(email:true, unique:true)
        }
}

Ela é mapeada para uma tabela no banco com as colunas id, version (adicionada automaticamente pelo Grails), name e email. Então, devemos criar um dataset como o abaixo: 

<?xml version="1.0" encoding="UTF-8"?>
<dataset>
        <person id="1" name="Pessoa 1" email="p1@email.com" version="1" />
        <person id="2" name="Pessoa 2" email="p2@email.com" version="1" />
        <person id="3" name="Pessoa 3" email="p3@email.com" version="1" />
        <person id="4" name="Pessoa 4" email="p4@email.com" version="1" />
        <person id="5" name="Pessoa 5" email="p5@email.com" version="1" />
</dataset>

Para que o plugin possa carregar corretamente esse dataset, é preciso seguir algumas convenções:

Onde colocar o dataset

O diretório no qual os datasets devem ser colocados é test/datasets: dentro desse diretório, serão criados subdiretórios que separarão cada conjunto de datasets para os testes. Por exemplo, se o teste especifica que usará o dataset "person", crie um diretório test/datasets/person para acomodar os arquivos xml do DBUnit.

Como indicar para a classe de teste quais datasets usar

Totalmente simples! Veja abaixo: 

class PersonTests extends GroovyTestCase {

        static datasets = ['person']

        void testCountPerson() {
                def c = Person.count()
                assertEquals 5, c
        }
}

A propriedade estática "datasets" é uma lista com todos os datasets necessários para aquele teste. No caso, todos os arquivos .xml dentro de test/datasets/person serão carregados. Se você salvar o xml descrito acima em test/datasets/person/data.xml o plugin se encarregará de carregar todos os dados descritos e os deixará disponíveis para o teste.

Dica: se você quiser, pode indicar a ordem na qual os arquivos serão carregados, numerando-os como a seguir:

  • test/datasets/person/0.data.xml
  • test/datasets/person/1.data.xml
  • test/datasets/person/2.data.xml

Note que a ordenação é lexicográfica.

Indicando mais de um dataset por teste

Basta adicionar à lista, todos os datasets desejados: 

static datasets = ['person', 'city', 'country']

Aqui a ordem de carga é de acordo com a ordem na qual os itens aparecem na lista. Ou seja, primeiro os arquivos em test/datasets/person/, depois test/datasets/city/ e assim por diante.

Datasets em "pacotes"

É possível separar os datasets em uma estrutura parecida com a de pacotes. Para a seguinte classe de teste: 

class PersonTests extends GroovyTestCase {

        static datasets = ['com/google/dbunit-plugin/person']

        void testCountPerson() {
                def c = Person.count()
                assertEquals 5, c
        }
}

O plugin navegará de modo a listar os arquivos ".xml" em test/datasets/com/google/dbunit-plugin/person/. Atenção para o fato de que a navegação NÃO é feita nos subdiretórios, apenas no diretório indicado.

Detalhes

O load/unload dos datasets para um teste acontecem apenas uma vez para cada classe de teste, da mesma maneira que a anotação @BeforeClass do JUnit. Funciona assim porque os teste de integração do grails são transacionais e então não é preciso "limpar" o ambiente para cada teste case (em uma release futura, o plugin irá decidir se precisa fazer isso se o teste não for transacional).

As operações executadas no load/unload dos datasets são, respectivamente, CLEAN_INSERT e DELETE.
